Case study for My Flix full-stack project

Overview

myFlix is a web app, developed using the MERN stack, that provides users with access to information about movies, directors, and genres. Users are able to create an account, update their personal data, and create a list of favorite movies

Purpose and Context

myFlix was a personal project I built as part of my web development course at CareerFoundry to demonstrate my mastery of full-stack JavaScript development.

Objective

The aim of the project was to have an ambitious full-stack project I can add to my professional portfolio. The problem I wanted to solve is to build the complete server-side and client-side for the application from scratch.

Approach

Back-end

For this project, I developed a RESTful API using Node.js and Express to interact with a MongoDB database. The API is designed to handle various HTTP methods like GET, POST, PUT, and DELETE, enabling clients to perform CRUD (Create, Read, Update, Delete) operations on a collection of movie information. The primary goal of the API is to serve and manage data about movies in JSON format.

Technology stack:

  • Node.js: Asynchronous, event-driven JavaScript runtime that handles multiple requests efficiently.
  • Express: Minimalist web framework for Node.js, which simplifies routing and middleware configuration.
  • MongoDB: A NoSQL, non-relational database that stores data in flexible, document-based structures (JSON-like BSON format). Perfect for applications where data structure is not fixed or pre-defined.

To test the functionality of the API, I used Postman, a popular API testing tool that allows for interaction with the API endpoints in an isolated environment. With Postman, I tested various API routes and scenarios to ensure the API handled requests and errors as expected. Finally, I also tested the authentication flows with both Basic HTTP and JWT authentication to confirm that unauthorized users could not access restricted endpoints.

Front-end

After completing the backend API, I proceeded to build the user interface to allow users to interact with the API. This interface is a single-page application (SPA) developed using React and React-Redux, designed to handle user requests and display responses from the server-side efficiently. The goal was to create a dynamic, responsive, and user-friendly web application that communicates seamlessly with the backend API.

Technology stack:

  • React: A powerful JavaScript library for building user interfaces, particularly well-suited for creating dynamic and interactive SPAs. React uses a component-based architecture, which allowed for better organization and reusability of UI elements.
  • React-Redux: A state management library that works in conjunction with React to handle application-wide state efficiently, making it easier to manage and update UI components based on changes in the data.
  • CSS: For styling the application, ensuring that the layout was both visually appealing and responsive across various devices and screen sizes.

The SPA is designed to allow users to search for, view, and manage movie information provided by the backend API. The user can select a movie to watch, enter this movie to his favorite list or modify their user data. By building this single-page application with React and React-Redux, I created a dynamic and responsive interface that seamlessly communicates with the backend API. The use of Redux for state management ensured smooth handling of data and user interactions across the application, while Reactâ€™s component-based architecture made the UI modular and maintainable. The SPA provides an intuitive and user-friendly experience for interacting with movie data, and it showcases how front-end technologies can be combined to build scalable, efficient web applications.

Challenges

This project was both the most rewarding and the most challenging I have undertaken so far. From designing the API and working with database structures to building a responsive client-side interface, I experienced a wide range of technical and conceptual learning curves. Ultimately, the project helped me grow as a developer, enhancing my skills in both backend and frontend development.

One of my favorite aspects of this project was building the backend. I enjoyed working with Node.js and Express to create a structured and efficient RESTful API. The flexibility of using MongoDB as a non-relational database allowed me to focus on creating a dynamic schema for movie information without being constrained by the rigid structure of relational databases.

Duration

Between the back-end and the front-end, I would say that this entire project took me around 20 days for completion.
